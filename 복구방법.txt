# 프로젝트 복구 방법

이 프로젝트는 USB 용량을 줄이기 위해 `node_modules` 폴더와 `package-lock.json` 파일이 삭제되었습니다. 프로젝트를 다시 실행하려면 다음 단계를 따라 필요한 의존성 파일들을 복구해야 합니다.

## 1. 필요한 도구 설치 (Node.js 및 npm)

아직 Node.js와 npm이 설치되어 있지 않다면, 먼저 설치해야 합니다.
공식 웹사이트에서 다운로드하여 설치할 수 있습니다: [https://nodejs.org/](https://nodejs.org/)

## 2. 의존성 파일 복구

각 프로젝트 디렉토리로 이동하여 `npm install` 명령어를 실행합니다. 이 명령어는 `package.json` 파일에 명시된 모든 의존성을 설치하고, `package-lock.json` 파일을 자동으로 생성합니다.

### 2.1. 프로젝트 루트 디렉토리 복구

1.  명령 프롬프트(CMD) 또는 터미널을 엽니다.
2.  이 프로젝트의 루트 디렉토리로 이동합니다. (예: `cd C:\Users\원용우\Desktop\Erum_이룸 - 복사본`)
3.  다음 명령어를 실행합니다:
    ```bash
    npm install
    ```

### 2.2. `client` 디렉토리 복구

1.  명령 프롬프트(CMD) 또는 터미널에서 `client` 디렉토리로 이동합니다. (예: `cd C:\Users\원용우\Desktop\Erum_이룸 - 복사본\client`)
2.  다음 명령어를 실행합니다:
    ```bash
    npm install
    ```

### 2.3. `server` 디렉토리 복구

1.  명령 프롬프트(CMD) 또는 터미널에서 `server` 디렉토리로 이동합니다. (예: `cd C:\Users\원용우\Desktop\Erum_이룸 - 복사본\server`)
2.  다음 명령어를 실행합니다:
    ```bash
    npm install
    ```

## 3. 프로젝트 실행

모든 의존성 파일이 복구되면, 각 프로젝트의 실행 지침에 따라 애플리케이션을 실행할 수 있습니다. 일반적으로 `client` 디렉토리에서는 프론트엔드를, `server` 디렉토리에서는 백엔드를 실행합니다.

예시:
-   `client` 디렉토리에서: `npm start` (또는 해당 프로젝트의 실행 명령어)
-   `server` 디렉토리에서: `node index.js` (또는 해당 프로젝트의 실행 명령어)

이 단계를 따르면 삭제된 파일들을 복구하고 프로젝트를 다시 정상적으로 실행할 수 있습니다.
